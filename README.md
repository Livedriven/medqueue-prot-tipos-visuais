# MedQueue-Prot-tipos-Visuais
Este repositório funciona como um **laboratório de prototipação visual** para o desenvolvimento da interface do sistema **MedQueue**.

O objetivo é construir, testar e validar a camada visual da aplicação de forma isolada, utilizando **HTML, CSS e JavaScript**, antes de integrar a interface à arquitetura principal do projeto, baseada em **MVC* com **Vanilla JavaScript**.

## 🎯 Objetivo do Repositório
Este projeto tem como foco o desenvolvimento estático e interativo da interface do MedQueue, permitindo validar:
* Estrutura visual das telas;
* Organização dos componentes da interface;
* Responsividade em diferentes tamanhos de tela;
* Acessibilidade básica com uso correto de elementos semânticos;
* Interações simples com JavaScript;
* Experiência do usuário antes da integração com a lógica principal do sistema.

## 🛠️ Tecnologias Utilizadas
* **HTML5:** Estruturação semântica e acessibilidade (labels e inputs).
* **CSS3:** Estilização usando CSS Grid, Flexbox e variáveis modulares.
* **JavaScript Vanilla:** Interações básicas (menus, modais e elementos ocultos).
* **Design Pattern:** Foco na separação de responsabilidades (View isolada).

## 📂 Estrutura do Projeto
* `index.html`: Estrutura principal contendo dashboard, menu e formulários.
* `css/`: Módulos de estilo (`reset.css`, `variable.css`, `style.css`).
* `js/`: Script de interações visuais básicas.
* `assets/`: Ícones e recursos visuais.

## ⚙️ Funcionalidades do Protótipo
* Dashboard principal do sistema.
* Menu de navegação (exibição e ocultação).
* Formulários modais sobrepostos.
* Organização visual dos cards de pacientes.

## 🚀 Como Visualizar
Como este é um protótipo front-end estático, basta clonar este repositório e abrir o arquivo `index.html` diretamente em qualquer navegador moderno.

## 🧪 Status e Próximos Passos
Atualmente em fase de prototipação visual. Os próximos passos incluem:
* Melhorar a responsividade.
* Validar regras de acessibilidade.
* Integrar a interface validada à arquitetura MVC do repositório principal.

Ambiente de prototipação isolado para a interface do sistema hospitalar **MedQueue**. O foco aqui é construir e validar a **Camada View** (UI/UX, responsividade e acessibilidade) antes da injeção dinâmica no motor **MVC/SPA** do projeto principal.

## 🛠️ Stack e Funcionalidades Visuais

| Tecnologia | Aplicação no Protótipo |
| :--- | :--- |
| **HTML5** | Semântica estrutural e acessibilidade de formulários. |
| **CSS3** | CSS Grid, Flexbox e componentização via variáveis. |
| **Vanilla JS** | Manipulação básica de DOM (Modais, Menus e Toggles). |
| **Design** | Dashboard de triagem, modais flutuantes e filtros de status. |

## 🚀 Como testar
Sem dependências complexas. Clone o repositório e abra o arquivo `index.html` no seu navegador (ou via extensão Live Server).

O código está modularizado separando estilos de reset, variáveis globais e o layout principal, mantendo a preparação exata para a futura componentização no JavaScript.
